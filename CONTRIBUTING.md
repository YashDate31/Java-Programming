# Contributing Guidelines

Thanks for your interest in improving this repository.

## Branching

- Create a feature branch from `main`
- Keep commits small and meaningful

## Coding Standards

- Use clear PascalCase class names
- Match filename with public class name
- Keep code simple and beginner-friendly
- Avoid unnecessary comments

## Pull Request Checklist

- Code compiles successfully
- No accidental `.class` commits
- Folder placement matches practical/unit
- README or mapping updated if structure changes

## Commit Message Style

- `feat: add Pr18 JComboBox demo`
- `fix: correct package declaration in Pr07`
- `docs: update migration map`
